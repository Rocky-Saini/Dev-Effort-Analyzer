Getting Started

3 simple steps to generate your first WBS & Impact Analysis

1

Install & Open

Download the app for your OS and launch it. No configuration needed.

# Linux AppImage (no install) chmod +x DevEffort-Analyzer.AppImage ./DevEffort-Analyzer.AppImage # Linux .deb sudo dpkg -i dev-effort-analyzer_amd64.deb # Windows — just double-click the .exe installer # macOS — drag to Applications from DMG
2

Prerequisites

Make sure these are installed on your machine:

✓
Git For branch management & repo access
✓
Kiro CLI AI engine that generates documents
✓
Kiro CLI Login Run kiro-cli login once to authenticate
3

Use the Tool

Inside the app:

① Add your project repo path
② Select branch (search by name or author)
③ Paste Jira story content
④ Add any extra context from your side
⑤ Click Generate — Kiro AI does the rest
⑥ Download WBS & Impact Analysis files

What You Get

Professional documents generated in your repo's development-effort/ folder

📋 WBS_FUL2-288698.md

• JIRA Ticket | Assignee | Total Efforts | DEV Date | UAT Date

• Detailed task breakdown with effort hours

• Buffer hours (Dev Testing, UAT, Impact Analysis)

• Branch & Commit Guidelines

• Dependencies & Regression Areas

📊 Impact_Analysis_FUL2-288698.md

• Ticket No & Short Task Description

• Short Impact Analysis (1-2 lines)

• Detailed Impact (APIs, DB, Events, Security)

• Upstream/Downstream Dependencies

• Regression Risks & Rollback Plan

Alternative: Run from Source

If you prefer not to install the desktop app

git clone https://github.com/Rocky-Saini/Dev-Effort-Analyzer.git cd Dev-Effort-Analyzer node server.js # Open http://localhost:3500 in browser

Requires: Node.js 18+, Git, Kiro CLI (authenticated)
